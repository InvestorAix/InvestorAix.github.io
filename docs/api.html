Documentação da API

Autenticação

Todas as requisições à API devem incluir um token de autenticação no cabeçalho:

Authorization: Bearer seu_token_aqui

Endpoints

GET /api/v1/portfolio

Retorna o portfólio do usuário.

Resposta

{
    "success": true,
    "data": {
        "total_value": 150000.00,
        "assets": [
            {
                "symbol": "AAPL",
                "quantity": 100,
                "current_price": 150.00,
                "total_value": 15000.00
            }
        ]
    }
}

POST /api/v1/analysis

Realiza análise de um ativo específico.

Request

{
    "symbol": "AAPL",
    "timeframe": "1d",
    "indicators": ["rsi", "macd"]
}

Resposta

{
    "success": true,
    "data": {
        "symbol": "AAPL",
        "analysis": {
            "rsi": 65.4,
            "macd": {
                "value": 2.5,
                "signal": 1.8
            },
            "recommendation": "HOLD"
        }
    }
}

GET /api/v1/market-data

Retorna dados de mercado em tempo real.

Parâmetros

  • symbol - Símbolo do ativo (obrigatório)
  • interval - Intervalo de tempo (1m, 5m, 15m, 1h, 1d)
  • limit - Número de registros (máximo 1000)

Códigos de Erro

Código Descrição
400 Requisição inválida
401 Não autorizado
403 Acesso proibido
404 Recurso não encontrado
429 Muitas requisições
500 Erro interno do servidor
